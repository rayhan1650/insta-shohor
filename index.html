Q: How does javascript worked?

Answer:

When a browser or server gets javascript code, it has to convert js code to machine-readable code because the machine can't understand js code directly. To convert js code to machine-readable code, browser or node.js server run time should have a mechanism. So every browser has a javascript engine to convert js code to machine-readable code. In Google Chrome JS engine name is the V8 engine that is also used in node.js runtime. In Firefox it is SpiderMonkey, in Internet Explorer it is Chakra, in Safari it is JavaScriptCore. Different browsers implemented JS engines in different ways according to ECMAScript specification. V8 engine takes js source code and by using Parser it makes an Abstract Syntax Tree. After that it read every line of code by Interpreter and it also use Compiler for boosting runtime performance and convert js code to machine code. Then it generate byte code and make an output into the display.

Q: What is the difference between local storage and session storage?

Answer:

# Local Storage Session Storage
1 Local storage used to store data for a long period so that user can get his previous changed data even if he restart the browser. Session storage used to store data temporarily and data is gone after tab closed.
2 Generally local storage capacity is 5MB or 10MB. Generally session storage capacity is 5MB.
3 In local storage data can be cleared by manually or by JS code. In session storage data can be cleared when a tab closed.
4 We can get local storage data by closing and restarting browser and also by restarting computer. We can get session storage data by refreshing the tab, but if we close the tab or restart the browser, we lose data.
5 In local storage only client can read data but server can't. Also in session storage only client can read data when session is available, but server can't.
